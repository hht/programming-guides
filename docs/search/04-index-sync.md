# 04 — 索引同步（写 → 可搜）

## 不变量

- 权威写成功后，索引按 INPUTS §4 策略更新；策略写进代码与测试，禁止「有时触发器有时脚本」双源。  
- **同事务**策略：提交后同一连接/下一读必须可搜到（读己之写）。  
- **异步**策略：必须公布 **可见延迟 SLA**（秒）；超时未可见 = 同步故障，进错误/告警分类（运维第三方可选，不进必勾）。  
- 删除/可见性降级：索引侧必须同步移除或标记不可搜，禁止幽灵命中。

## 步骤规格（实现自写）

1. 选定 INPUTS §4 三种之一并只实现一种主路径。  
2. **同事务触发器/生成列**：写业务行 → `search_vector` 更新同事务；COMMIT 后可读。  
3. **应用双写同事务**：业务 UPDATE + 索引列 UPDATE 同一事务；任一步失败 → ROLLBACK（对齐 postgres `05`）。  
4. **异步 outbox**：业务事务写 outbox；worker 建索引；记录 `indexed_at`；探针在 SLA 内轮询可见。  
5. 回填：历史数据一次性 backfill 迁移/作业；作业幂等（按主键 upsert）。  
6. 专用引擎（若启用）：以 outbox/CDC 为默认；禁「只改引擎不改 PG 权威」。

## 失败分类 / 默认值

| 情况 | 行为 |
|------|------|
| 同事务索引写失败 | 整笔业务写失败 |
| 异步超 SLA | `search.index_lag`（或内部码）；用户查询仍可返回陈旧/空，但须可观测 |
| 删除后仍命中 | **缺陷**：同步路径 FAIL |

## 单测探针（case → 期望）

| case | 期望 |
|------|------|
| 插入后立即搜（同事务策略） | 命中 |
| 删除后搜 | 不命中 |
| 异步：写入后 t < SLA | 最终命中；测试可注入加速 worker |
| 双写第二步失败 | 无业务脏行、无半新索引 |
