# 00 — 原则与不变量

## 决策优先级

正确性（可回滚的成功发布）> 可验证性 > 简洁性 > 复用 > 速度。

## 硬不变量

1. **密钥不入库**：仓内仅变量名与注入点；值在 Secrets/主机。  
2. **不可变镜像**：发布物 = 带 digest/tag 的镜像；禁「服务器上 git pull 当发布」。  
3. **健康门闸**：新版本未过健康检查不得标成功。  
4. **可回滚**：保留上一成功版本；失败默认自动或一键回滚。  
5. **环境成对**：staging/prod 配置结构同构，值不同。  
6. **deletion-first**：无 INPUTS 的环境/流水线不做。

## 超越

1. `对照：B 中更弱/未见「健康门闸未过不得标成功」硬门闸 → 本指南要求`  
2. `对照：B 中更弱/未见「上一版本可切回作为默认回滚」硬门闸 → 本指南要求`
