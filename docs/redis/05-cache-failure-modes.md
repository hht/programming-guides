# 05 — 缓存失败模式（穿透 / 击穿 / 雪崩）

## 不变量

- 三种模式**必须**在 INPUTS/实现中有对策；默认如下表，禁止「上线再看」。 
- 对策不得破坏 `04` 写序（源先于缓存）。

## 模式与默认对策

| 模式 | 含义 | 默认对策 |
|------|------|------------------|
| **穿透** | 查不存在的 id，反复打源 | **默认短 TTL 空值占位**（INPUTS §11，默认 30–60s）。布隆过滤仅当 INPUTS §12 勾选「布隆」；**禁止**「或」双默认 |
| **击穿** | 热点键过期瞬间并发打源 | **single-flight / 互斥回填**：同 key 仅一请求回源，其余等或短暂重读缓存（INPUTS §12） |
| **雪崩** | 大量键同时过期 | TTL **加抖动**：`ttl = base + random(0..jitter)`；jitter 默认 `min(60, base*0.1)` 秒（可 INPUTS 改） |

## 步骤规格（实现自写）

1. **穿透** 
 - 源返回空 → `SET key empty_marker EX short_ttl`（marker 与真值可区分）→ 调用方得「不存在」。 
 - 禁止对不存在键「不写缓存」却无其它过滤。 
2. **击穿** 
 - 未命中时尝试获取 per-key 回填锁（可用 `06` 短 TTL 锁，或进程内 single-flight）。 
 - 未抢到锁：短暂 sleep/重试 GET；**默认最多 5 次、总等待 ≤ 200ms**（改则写入 INPUTS §12 数字）；超时 → **`STORE_UNAVAILABLE`**（不直读源，除非 §8b 勾降级）。禁止无界打源。 
3. **雪崩** 
 - 所有批量回填/预热 SET 使用抖动 TTL。 
 - 禁把整库键设成同一绝对过期时间戳。 
4. **逻辑过期（可选）** 
 - value 内嵌 `expire_at`；物理 TTL 更长；过期后单飞刷新——仅当 INPUTS 勾选。

## 失败分类 / 默认值

| 情况 | 行为 |
|------|------|
| 空值占位命中 | 直接「不存在」，不打源 |
| 回填锁未获得 | 重试 GET（有界次数）；**超时默认 `STORE_UNAVAILABLE`**。仅当 INPUTS §8b 勾选「降级直读」时才直读源；**禁止**规格内未勾选的「或降级」 |
| 抖动后 TTL | 仍 > 0 且 ≤ INPUTS 上限 |

## 单测探针（case → 期望）

| case | 期望 |
|------|------|
| 连续查不存在 id | 源调用次数有界（占位生效） |
| 热点过期并发 N | 回源次数 ≪ N（单飞/互斥） |
| 批量 SET | TTL 不全相等（抖动） |
