# 多 Agent 对抗式审核 — 最终方案（SSOT）

> **本文件是对抗 LOOP 的唯一操作规格。** 
> 元指南流程入口：[how-to-write-guides.md](./how-to-write-guides.md) §3.6（指向本文）。 
> 门槛仍以元指南 **§1.2–1.3** 为准；本文只规定**怎么审、怎么裁、怎么收工**。

---

## 0. 一句话

五个只读 agent 各守一角，判定「仅凭领域指南 + 合格 INPUTS 能否自写达标」；主 agent 仲裁真伪 BLOCKER → **只改规格** → 全新五路，直到 **5/5 PASS**。

---

## 1. 何时开跑（触发矩阵）

| 触发 | 范围 | 通过线 |
|------|------|--------|
| **新品类**首次标 DONE | 该域全五角 | 必须 5/5 |
| **核心路径 / INPUTS / commands / 02 词表 / 11** 实质变更后重审 | 该域全五角 | 必须 5/5 才能继续宣称 DONE |
| 仅文案对齐、状态 SSOT、无行为变更的措辞 | **可不开** | — |
| 用户明示「对抗 / 重审」 | 用户指定域；默认全五角 | 5/5 |
| 聚焦预检（可选，**不得**单独标 DONE） | 仅 B+E 或用户点名角 | 预检通过 ≠ DONE |

**禁止**：用「上次 PASS + 只改厚了」自动继承 5/5。实质改规格 → 必须重开。

---

## 2. 固定配置

| 项 | 值 |
|----|-----|
| 角色数 | **5**（A–E）；**禁止**一人多角、禁止合并视角 |
| 模型 | 默认 **`grok-4.5-fast-xhigh`**；宿主拒 slug → 用可用列表中最接近的 grok/fast 高档，**本轮仲裁 NOTES 写明实际 model** |
| 模式 | `readonly=true`；每角 **新开** Task；**禁止 resume 审同一轮** |
| 通过 | **SCORE = 5/5**；≤4/5 = 未完成 |
| FAIL 处理 | **只改规格**；禁止往指南塞业务实现消 FAIL；禁止改元门槛来就范 |
| 轮次 | 无硬上限；**同一真 BLOCKER 连续 2 轮未消** → 停手升级人工 |
| 语言 | 子 agent 输出 **≤300 字中文**；主裁 **≤150 字** |

### 五角

| 角 | 深挖 |
|----|------|
| **A** | Agent 执行协议 / 闭环（README 步骤 → INPUTS 停写 → commands 收口） |
| **B** | 核心正确性路径规格（步骤、失败分类、默认值、探针） |
| **C** | INPUTS / 契约 / 环境或地址等价 SSOT |
| **D** | 目录 · 业务命名/词表 · UI 状态交付（无 UI 品类须显式 N/A） |
| **E** | 测试矩阵 / commands / acceptance |

---

## 3. 判定句（唯一；写入每个 prompt）

```text
实现 agent 仅凭领域指南 + 合格 INPUTS，
能否在新仓自写达到元指南 §1.2–1.3。

指南≠项目；缺业务源码≠FAIL；规格不清导致必瞎猜=FAIL。
运维第三方不进必勾。禁止自有仓当唯一标杆。禁止要求往指南塞业务实现。
交付验收默认勾 11 的 A+B+D；11-C（§1.3 元判定）仅指南维护者 — 不得因「实现 agent 勾不了 C」判 E 失败（除非 commands 错误强制交付勾 C）。
```

## 4. 开跑步骤（宿主）

### 4.1 前置

- [ ] 领域目录按元指南 §4 齐套（含 templates 仅允许类） 
- [ ] 新品类：§7 清单 1–6 已勾 

### 4.2 发起一轮（ROUND = n）

1. **同一条消息内并行**发起 5 个 Task（理想路径）。 
2. 每个 Task：`model` 同上；`readonly=true`；不传 `resume`；`subagent_type=generalPurpose`（或宿主等价）。 
3. Prompt = §6 骨架，只替换：绝对路径、`<domain>`、视角字母。 
4. **收齐 5 份合法 `VERDICT` 后**主 agent 才仲裁（见 §4.3 缺角规则）。 

### 4.3 缺角 / 连接失败（实操写明）

| 情况 | 动作 |
|------|------|
| 同轮缺 1–k 角（超时、连接断开、无 VERDICT） | **ROUND 号不变**；只补跑缺角；补齐后再裁 |
| 同轮补跑仍缺角，累计补跑 ≥2 次 | 改 **串行** 逐角开跑；仍不成 → 升级人工 / 换 model |
| 某角输出无 `VERDICT:` 或不分辨 PASS/FAIL | 该角作废，补跑 |
| **禁止** | 用 4/5 或「其余都 PASS」宣布 DONE |

### 4.4 仲裁（主 agent）

输出（≤150 字）：

```text
DOMAIN: <domain>
ROUND: <n>
SCORE: <k>/5
FAILS: <字母列表或 none>
ACTION: DONE | FIX→再开5路
NOTES: <真 BLOCKER 一句；假阳性一句可空；实际 model 若非默认则写明>
```

### 4.5 真伪 BLOCKER

| 真（必须改规格） | 假阳性（记 GAPS，不挡 5/5） |
|------------------|------------------------------|
| 文件间矛盾 / 双 SSOT | 「希望更长」但已有默认可写 |
| 关键决策未写明 → 必瞎猜 | 缺可运行业务源码 |
| commands 与 INPUTS/05 冲突 | 运维第三方未进必勾 |
| 交付 `check` 误强制勾 11-C | 工具版本「写稳定版」类可自写细节 |
| GAPS 造成跨文件矛盾 | 升格为真 BLOCKER |

真 → **只改规格** → `ROUND=n+1` **全新 5 路**（禁止 resume；禁止只重跑 FAIL 角冒充新轮）。 
GAPS 不阻塞 DONE；可列进后续季检。

### 4.6 收工

- 5/5 后：勾元指南 §7 对应项（新品类）或更新该域维护记录一行（重审）。 
- 提交：`docs(<domain>): <why>`；多域可 `docs: adversarial PASS for a,b,c`。 
- **可选**薄归档（禁止过程流水账入库）：在该域 `sources.md` 末追加：

```markdown
## 对抗

| 日期 | ROUND | SCORE | model |
|------|-------|-------|-------|
| YYYY-MM-DD | final | 5/5 | grok-4.5-fast-xhigh |
```

## 5. 范围变体

| 变体 | 用法 |
|------|------|
| **Full** | 默认；标 DONE / 实质变更后唯一合法变体 |
| **Focused** | 仅 B+E 或点名角；只作预检；报告须写 `VARIANT: focused`；**不得**改 roadmap DONE |
| **多域并行** | 允许多域同时开 Full；每域独立 ROUND/SCORE；主机资源不足时 **按域串行、域内五角并行** |

## 6. 粘贴用 prompt 骨架

```text
必读（只读）：
1) 元指南门槛：<绝对路径>/docs/meta/how-to-write-guides.md（§1.2–1.3）
2) 对抗方案：<绝对路径>/docs/meta/adversarial-review.md（判定句与真伪表）
3) 领域指南：<绝对路径>/docs/<domain>/（含 templates 允许文件）

判定（唯一）：实现 agent 仅凭领域指南 + 合格 INPUTS，
能否在新仓自写达到元指南 §1.2–1.3。
指南≠项目；缺业务源码≠FAIL；规格不清导致必瞎猜=FAIL。
运维第三方不进必勾。禁止自有仓当唯一标杆。禁止要求往指南塞业务实现。
交付验收默认 11 的 A+B+D；11-C 仅维护者（除非本域 commands 错误强制交付勾 C → 可 FAIL）。

视角：<A|B|C|D|E>（只深挖该角；仍须对照 §1.2–1.3）

≤300字中文，严格：
VERDICT: PASS 或 FAIL
BLOCKERS: （FAIL 列 1–5 条；PASS 写 none）
GAPS: 最多 3 条非阻塞
```

## 7. 与本仓其它契约的关系

| 契约 | 关系 |
|------|------|
| 元指南 §1.2–1.3 | **门槛 SSOT**（审什么） |
| 本文 | **对抗操作 SSOT**（怎么审） |
| 各域 `11` A+B+D / C | 交付 vs 维护者；E 角须遵守 |
| 全局 agent Contract | 无用户明示不 commit；对抗中只读子 agent 不写盘 |
| 语言工程薄规（若未来增加） | 同用 Full 对抗；禁止另发明第二套五角 |

---

## 8. 反模式（禁止）

1. 4/5 宣称 DONE或「差不多」 
2. 用加示例业务代码消 FAIL 
3. resume 同一子 agent 做下一轮 
4. 合并 A–E 一次问完 
5. 把 GAPS 当 BLOCKER 无限加厚注水 
6. 假阳性当真（缺源码、要 Sentry） 
7. 过程 ROUND 长文进领域正文（仅允许 `sources` 一行表） 

---

## 9. 最小检查清单（主 agent 开跑前）

- [ ] 触发类型已选（新品类 / 重审 / 预检） 
- [ ] 域路径绝对路径正确 
- [ ] 五角 prompt 视角字母互不重复 
- [ ] readonly、无 resume、model 已定 
- [ ] 已知：缺角规则与真伪表 
- [ ] 收工：5/5 → 归档一行（可选）+ 用户若要求再 commit
