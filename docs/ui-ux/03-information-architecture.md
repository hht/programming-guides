# 03 — 信息架构

## 不变量

- 先任务后屏幕；禁先堆组件  
- 导航深度默认 ≤3；更深须在 INPUTS 说明  

## 步骤规格

1. 写出 **用户流**：起点 → 决策点 → 成功 / 失败出口（每种出口一句话）→ `design/flows/<task-id>.md`。  
2. 列出 **屏幕清单**（`screen-id` + 目的）；每屏标注主 CTA → INPUTS §3b + `design/screens/<screen-id>.md`。  
3. 定义 **导航模式**：一律按 INPUTS §2b 已选项（禁止另开第三种壳）。  
4. 破坏性操作：须二次确认或 undo 路径（钉一种，**默认二次确认**）。  

## 失败分类

| 情况 | 行为 |
|------|------|
| 任务含糊 | INPUTS BLOCKED |
| 两屏主 CTA 竞争 | 合并或降级其一 |

## 单测探针（设计评审）

| case | 期望 |
|------|------|
| 外人只看流图 | 能说出成功标准 |
| 每屏 | 恰有一个主 CTA |
| screen-id | 与 `02` join key 一致 |
